pub struct SeriesDivide {
    tag_columns: Vec<String>,

    input: LogicalPlan,

Fields§

§tag_columns: Vec<String>§time_index_column: String

SeriesDivide requires time_index column’s name to generate ordering requirement for input data. But this plan itself doesn’t depend on the ordering of time index column. This is for follow on plans like RangeManipulate. Because requiring ordering here can avoid unnecessary sort in follow on plans.
