Buffer of plugin option tags that were dropped during config loading because they are not recognized by the current build.

Config loading typically happens before the global tracing subscriber is installed, so a warn! emitted at that point is silently lost. Instead we buffer the dropped tags here and let the server flush them (via take_dropped_plugin_warnings) once logging is initialized, so the warning actually reaches the configured log output.
