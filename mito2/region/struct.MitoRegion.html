Expand description

Metadata and runtime status of a region.

Writing and reading a region follow a single-writer-multi-reader rule:

  • Only the region worker thread this region belongs to can modify the metadata.
  • Multiple reader threads are allowed to read a specific version of a region.

Fields§

§region_id: RegionId

Id of this region.

Accessing region id from the version control is inconvenient so we also store it here.

§version_control: Arc<VersionControl>

Version controller for this region.

We MUST update the version control inside the write lock of the region manifest manager.

§access_layer: AccessLayerRef

SSTs accessor for this region.

§manifest_ctx: Arc<ManifestContext>

Context to maintain manifest for this region.

§file_purger: FilePurgerRef

SST file purger.

§provider: Provider

The provider of log store.

§last_flush_millis: AtomicI64

Last flush time in millis.

§last_compaction_millis: AtomicI64

Last compaction time in millis.

§time_provider: Arc<dyn TimeProvider>

Provider to get current time.

§topic_latest_entry_id: AtomicU64

The topic’s latest entry id since the region’s last flushing. Only used for remote WAL pruning.

The value will be updated to the latest offset of the topic if region receives a flush request or schedules a periodic flush task and the region’s memtable is empty.

There are no WAL entries in range [flushed_entry_id, topic_latest_entry_id] for current region, which means these WAL entries maybe able to be pruned up to topic_latest_entry_id.

§written_bytes: Arc<AtomicU64>

The total bytes written to the region.

§staging_partition_info: Mutex<Option<StagingPartitionInfo>>

Partition info of the region in staging mode.

During the staging mode, the region metadata in VersionControlRef is not updated, so we need to store the partition info separately.

§stats: ManifestStats

manifest stats
